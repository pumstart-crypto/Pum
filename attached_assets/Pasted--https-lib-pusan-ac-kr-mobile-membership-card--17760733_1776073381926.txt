제공해주신 부산대학교 도서관 모바일 이용증 페이지(`https://lib.pusan.ac.kr/mobile/membership-card`)를 분석하여, 제작 중이신 앱에서 동일한 기능을 구현하기 위한 기술적 절차와 핵심 로직을 정리해 드립니다.

### 1. 시스템 구조 분석

이 페이지는 단순히 이미지를 보여주는 것이 아니라, **사용자의 인증 상태(세션)를 확인한 뒤 실시간으로 QR 코드를 생성**하는 동적 페이지입니다.

* **인증 방식:** 부산대학교 도서관은 보통 학번(ID)과 비밀번호를 통한 세션 방식 또는 OAuth2 기반의 통합 인증(SSO)을 사용합니다.
* **QR 데이터:** QR 코드 안에는 보통 `학번`, `생성 시간`, `암호화된 토큰` 등이 포함됩니다. 보안을 위해 일정 시간마다 QR이 갱신되는 'Dynamic QR' 방식일 가능성이 높습니다.

---

### 2. 구현을 위한 기술적 단계

직접 만드시는 앱에서 이 기능을 구현하려면 다음과 같은 흐름이 필요합니다.



#### 단계 1: 인증 (Login)
사용자가 입력한 아이디와 비밀번호를 부산대학교 도서관 서버로 전송하여 인증을 받아야 합니다.
* **방법:** `POST` 요청을 통해 로그인 API(또는 로그인 처리 페이지)에 데이터를 보냅니다.
* **결과:** 인증에 성공하면 서버로부터 `Cookie`(예: JSESSIONID) 또는 `Token`을 받게 됩니다. 이 세션 정보를 유지해야만 QR 생성 페이지에 접근할 수 있습니다.

#### 단계 2: QR 데이터 추출 (Scraping 또는 API 호출)
로그인된 상태(세션 유지)로 `https://lib.pusan.ac.kr/mobile/membership-card` 페이지의 소스를 읽어옵니다.
* **분석 포인트:** HTML 소스 내에서 `<img>` 태그의 `src` 주소를 찾거나, QR 생성을 위해 호출하는 JavaScript 함수를 찾아야 합니다.
* **데이터 형태:** * 서버에서 이미지 자체를 생성해 주는 경우: 해당 이미지 URL을 로드.
    * 텍스트 데이터를 주는 경우: 앱 내부 라이브러리(예: `react-native-qrcode-svg`, `ZXing` 등)를 이용해 직접 QR 생성.

#### 단계 3: 앱 내 표시 및 갱신
* **보안 유지:** 인증이 완료된 사용자에게만 QR 화면을 노출하도록 분기 처리를 합니다.
* **유효 시간:** 만약 QR 코드가 시간에 따라 변한다면(TOTP 방식), 주기적으로(예: 30초마다) 데이터를 다시 불러와서 QR을 갱신하는 로직이 필요합니다.

---

### 3. 개발 시 고려해야 할 핵심 사항

1.  **세션 관리 (Session Management):**
    웹 브라우저가 아닌 앱 환경에서 구현할 경우, HTTP 클라이언트(예: Axios, Fetch)가 쿠키를 자동으로 관리하도록 설정하거나, 수동으로 `Set-Cookie` 헤더를 처리해야 합니다.

2.  **WebView 활용 (가장 간단한 방법):**
    직접 데이터를 파싱하는 것이 어렵다면, 앱 내부에 **WebView**를 띄우는 방식이 있습니다.
    * 로그인 페이지를 WebView로 띄움 → 사용자 로그인 → 성공 시 QR 페이지로 리다이렉트 → 필요 없는 UI(헤더, 푸터)를 CSS 주입(JavaScript Injection)으로 숨김.

3.  **보안 및 정책 (중요):**
    * **개인정보:** 학번과 비밀번호를 앱 서버에 저장하지 말고, 학교 서버로 직접 전달하는 통로 역할만 하도록 설계해야 합니다.
    * **학교 정책:** 학교 측에서 외부 앱의 접근을 차단하기 위해 캡차(CAPTCHA)를 도입하거나 User-Agent를 체크할 수 있습니다.

### 요약된 구현 로직 (Pseudo Code)

```javascript
// 1. 로그인 요청
const session = await loginToPNU(id, pw);

if (session.isAuthenticated) {
    // 2. 인증된 세션으로 QR 페이지 데이터 요청
    const qrData = await fetchQRData(session.cookie);
    
    // 3. QR 생성 및 화면 표시
    renderQRCode(qrData);
} else {
    showErrorMessage("인증에 실패했습니다.");
}
```

이 분석 내용이 앱 개발 방향을 잡는 데 도움이 되기를 바랍니다. 추가로 특정 프로그래밍 언어(Java/Kotlin, Swift, React Native 등)에 맞는 코드가 필요하시면 말씀해 주세요.
