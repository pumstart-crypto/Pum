공개적으로 확인되는 자료만 기준으로 보면, **PNU Blace는 부산대 도서관 좌석 시스템을 새로 만든 게 아니라, 공식 좌석 시스템의 로그인·조회·좌석 조작 흐름을 서버에서 대신 수행한 뒤 자체 웹 UI와 부가 기능을 얹은 형태**로 보입니다. PNU Blace 자체도 부산대학교 공식 서비스가 아닌 “개인이 개발한 비공식 서비스”라고 밝히고 있고, 로그인 화면에는 부산대 계정을 사용한다고 안내합니다. 반면 부산대의 `pyxis-api` 안내 페이지는 해당 호출이 “PYXIS 프로그램에서만 허가”된다고 표시하고 있어, 일반적인 공개 오픈 API처럼 바로 쓰는 구조는 아니었던 것으로 보입니다. ([pnu-blace.com][1])

구현 출발점은 **모바일 앱/웹뷰 트래픽 분석**이었던 것으로 보입니다. 공개 글에서는 도서관 좌석 앱 화면이 웹뷰처럼 보여서, 웹 버전이 없음을 확인한 뒤 아이폰 네트워크 프록시(찰스 프록시)로 요청 흐름을 분석했다고 적었습니다. 그 글에는 초기 버전 체크 → 세션 쿠키 발급 → 사용자별 키 발급 → 로그인 흐름을 파악했고, 이후 내 좌석 정보 조회와 좌석 관련 동작 가능 여부를 확인했다고 나옵니다. ([Velog][2])

그다음 서비스 구조는 **프론트 + 자체 백엔드 프록시 + 자체 DB** 조합입니다. 공개 설계 글에는 백엔드를 NestJS·TypeORM·PostgreSQL 기반 모놀리식 서버로 만들고, 서비스 인증은 “인증 위임 방식”으로 처리했다고 적혀 있습니다. 즉, 사용자의 학번/비밀번호로 학교 쪽 다단계 인증 절차를 서버가 대신 수행하고, PNU Blace 쪽 프론트엔드는 학교 세션을 직접 다루지 않고 서비스 전용 JWT로 동작하는 구조입니다. 동시에 로그인 페이지와 개인정보처리방침에는 비밀번호를 저장하지 않고 부산대 포털 인증 연동을 사용한다고 안내합니다. ([Velog][3])

열람실의 **자리 조회·선택·반납·연장**은 공개 글 기준으로 두 방식이 섞여 있습니다. 첫째, 전체 좌석 현황은 학교 쪽 `seatMap` 계열 응답을 받아 **HTML을 파싱/스크레이핑해서 JSON으로 변환**해 자체 프론트가 쓰도록 만들었습니다. 둘째, 좌석 지정·반납·연장 같은 조작은 학교 쪽 좌석 액션 API로 **그대로 프록시 전달**하는 방식이라고 설명합니다. 현재 좌석 상태 확인도 학교 API 조회를 그대로 활용한다고 적혀 있습니다. 즉, 좌석 로직 자체를 재구현한 것이 아니라, **공식 시스템의 응답을 읽기 쉬운 형태로 바꾸고 조작 요청은 중계**한 셈입니다. ([Velog][3])

UI 쪽도 공식 시스템을 완전히 베낀 게 아니라, **공식 좌석 배치 이미지는 활용하되 좌석 버튼은 자체 HTML 요소로 다시 그린** 흔적이 보입니다. 공개 글에는 원래 부산대 Place가 열람실 이미지를 배경으로 두고 자리마다 요소를 얹는 방식이라, PNU Blace도 배경 이미지는 유지하되 좌석 상태 표시는 자체 HTML 요소로 바꿔 웹에서 보기 좋게 구성했다고 적었습니다. 그래서 “자리 배치”는 공식 시스템 데이터를 따라가지만, 화면 경험은 별도 웹 앱으로 재구성한 형태입니다. ([Velog][3])

그리고 PNU Blace의 차별점인 **자동 예약, 알림, 통계, 예측**은 공식 사이트에서 직접 가져온 기능이 아니라 **자체 DB와 스케줄러로 덧붙인 기능**으로 보입니다. 공개 설계 글에는 크론 작업이 주기적으로 좌석 맵을 수집해 상태 변화를 저장하고, 그 로그로 빈자리 알림·개인 이용 통계·좌석 이용 패턴 예측을 제공한다고 적혀 있습니다. 이용약관과 개인정보처리방침에도 실시간 모니터링, 자동 예약, 알림, 통계, 좌석 예약 이력 처리 등이 명시돼 있습니다. ([Velog][3])

한 줄로 정리하면, **공식 부산대 도서관 좌석 시스템의 비공개/제한적 호출 흐름을 역분석해서, 자체 백엔드가 학교 로그인과 좌석 API를 대신 처리하는 프록시 계층을 만들고, 그 위에 웹 UI·DB·스케줄러·통계/알림 기능을 얹은 구조**입니다. 다만 내부 소스코드를 직접 본 것은 아니어서, 여기까지는 공개된 서비스 문구와 공개 글을 바탕으로 한 가장 타당한 재구성입니다. ([lib.pusan.ac.kr][4])
