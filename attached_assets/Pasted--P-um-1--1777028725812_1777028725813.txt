계정 탈퇴 시 처리 현황
현재 구현 상태 (⚠️ 주의):

DB 스키마 상 sessions 테이블은 users에 onDelete: 'cascade' 연결 — 사용자 삭제 시 세션 자동 삭제
그러나 서버 측 /auth/withdraw API 엔드포인트가 구현되지 않은 상태
웹 클라이언트에서는 localStorage 키 일괄 삭제만 수행 (서버 DB 데이터는 미삭제)
법적 요건 충족을 위해 추가 구현이 필요한 사항:

DELETE /auth/withdraw API 구현 — 사용자 레코드 삭제 시 관련 게시글·댓글·재정·성적 데이터 cascade 또는 명시적 삭제
학생증 이미지 삭제 처리 (현재 studentIdImageUrl이 DB에 영구 보관)
탈퇴 후 일정 기간 내 완전 파기 또는 익명화 처리 로직
이 명세를 바탕으로 개인정보 처리방침 문서를 작성하거나, 탈퇴 API를 구현하는 것도 도와드릴 수 있습니다.
