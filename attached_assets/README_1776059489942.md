# PNU Seat Proxy — 배포 가이드

## Oracle Cloud Seoul 무료 티어 배포

### 1. 서버 준비
```bash
# Oracle Cloud에서 VM 생성 후 SSH 접속
ssh ubuntu@<서버IP>

# Node.js 설치
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 방화벽 포트 오픈
sudo iptables -I INPUT -p tcp --dport 3000 -j ACCEPT
```

### 2. 코드 배포
```bash
mkdir ~/pnu-seat-proxy && cd ~/pnu-seat-proxy
# server.js, package.json 업로드

npm install
```

### 3. PM2로 상시 실행
```bash
sudo npm install -g pm2
pm2 start server.js --name pnu-proxy
pm2 save
pm2 startup  # 재부팅 시 자동 시작
```

### 4. Nginx 리버스 프록시 (선택)
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 사용법

### 좌석 조회
```bash
# loginId:password를 Base64로 인코딩
TOKEN=$(echo -n "학번:비밀번호" | base64)

# 새벽별당[24h]-B (roomId=8) 좌석 조회
curl -H "Authorization: Bearer $TOKEN" \
  "http://<서버IP>:3000/seat-room-seats?seatRoomId=8"
```

### 헬스체크
```bash
curl http://<서버IP>:3000/health
```

### 열람실 목록
```bash
curl http://<서버IP>:3000/rooms
```

## 핵심 포인트

- `redirect: "manual"` — 302 응답에서 JSESSIONID를 캡처하기 위해 리다이렉트를 수동 처리
- 세션 캐시 30분 — 매 요청마다 로그인하지 않음
- `needLogin` 발생 시 자동 재로그인 1회 시도
- Node 18+ 필수 (내장 fetch 사용)
